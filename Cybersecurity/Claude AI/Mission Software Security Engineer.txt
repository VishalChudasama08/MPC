== PHASE 1: Cybersecurity Fundamentals & Network Security ==
==================================================
- Goal: Understand how networks/systems actually get attacked before defending them

1. Networking basics for security
- OSI model, TCP/IP, ports, protocols (HTTP/HTTPS, DNS, FTP, SSH)
- Firewalls, VPNs, proxies, NAT
- source:- Professor Messer (CompTIA Network+ / Security+ playlists, YouTube)

2. Core security concepts
- CIA Triad (Confidentiality, Integrity, Availability)
- Authentication vs Authorization
- Encryption basics: symmetric vs asymmetric, hashing, salting
- TLS/SSL handshake

3. Certification target
- CompTIA Security+ (industry recognized, good for resume)
- source:- Professor Messer free videos + practice exams

== PHASE 2: Ethical Hacking & Penetration Testing ==
==================================================
- Goal: Learn to think like an attacker to defend like an engineer

1. OWASP Top 10 (MOST IMPORTANT - maps directly to your Java/PHP projects)
- SQL Injection -> test on your own Movie_Booking_System_in_PHP
- Broken Authentication -> test on your own KeepNotes_SpringBoot login
- XSS (Cross Site Scripting)
- CSRF (Cross Site Request Forgery)
- Insecure Deserialization
- Security Misconfiguration
- source:- owasp.org/www-project-top-ten

2. Hands-on labs (FREE)
- TryHackMe - "Pre Security" and "Web Fundamentals" paths
- PortSwigger Web Security Academy (FREE, best resource for web app pentesting)
- DVWA (Damn Vulnerable Web Application) - practice attacks legally on your own machine
- bWAPP - another vulnerable app for practice

3. Tools to learn
- Burp Suite (intercept/modify HTTP requests)
- OWASP ZAP (free alternative to Burp)
- Nmap (network scanning)
- SQLMap (automated SQL injection testing)

4. Certification target (after fundamentals solid)
- eJPT (entry level, practical, affordable)
- Later: OSCP (gold standard, do after job experience)

== PHASE 3: Digital Forensics & Incident Response ==
==================================================
- Goal: Understand what happens AFTER a breach - investigation and response

1. Core concepts
- Chain of custody
- Evidence acquisition (disk imaging, memory dumps)
- Log analysis (Windows Event Logs, Linux syslogs)
- Incident Response lifecycle: Preparation -> Detection -> Containment -> Eradication -> Recovery -> Lessons Learned

2. Tools
- Autopsy (open source forensic tool)
- Wireshark (network traffic analysis)
- Volatility (memory forensics)

3. Hands-on
- TryHackMe - "Digital Forensics" room series
- CyberDefenders.org - free DFIR challenges

== PHASE 4: Secure Coding (THIS IS YOUR EDGE) ==
==================================================
- Goal: Apply security directly to your existing Java/Spring skills
- This is what makes you "Software Security Engineer" not just "Security Analyst"

1. Secure coding in Java/Spring Boot
- Input validation (never trust user input)
- PreparedStatement vs raw SQL queries (you already use this - good habit)
- Spring Security: proper password hashing (BCrypt), session management
- JWT security best practices
- Dependency vulnerability scanning (OWASP Dependency-Check)

2. Practical project ideas (use for Software Project - I)
=> Build a vulnerability scanner for Spring Boot REST APIs
=> Take your own KeepNotes_SpringBoot and do a full security audit:
   - Find every input field, test for injection
   - Check password storage method
   - Check session handling
   - Document findings + fixes in a report
=> Build a secure login system demonstrating: rate limiting, account lockout,
   password hashing, CSRF protection, secure session cookies

3. Resources
- OWASP Cheat Sheet Series (cheatsheetseries.owasp.org)
- "Secure by Design" book (Manning) - long term read

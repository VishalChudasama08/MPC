Servlet Life Cycle

1. Loading and Instantiation

-- First tomcat server started
-- java files (servlet file, then extend HttpServlet and other java files) will be compiled and class files will be created
-- Then class file will be loaded in the memory
-- and servlet object is created (servlet class file object)

2. Initialization

-- call init() method.
-- servlet object will be initialized by invoking the init() method

3. Request Handling

-- call service() method.
-- it will handle Client Request.
-- if Client Request is get method then service() method calls the doGet() method.
-- if Client Request is post method then service() method calls the doPost() method.
-- Client Request sends it, it is created a new thread and then this thread handles the request (calls service() method and this method resolves the client request).
-- Then Client sends another request, this creates a new thread that handles the client request... this process continuously runs as per client request
-- it is a multi-threading process

4. Destroy

-- server stopped and closed database connection (if connected!) then calls destroy() method.
-- destroy() method deletes the servlet object

Note: steps 1, 2, and 4 run only one time. The init() and destroy() methods are called one time, but step 3 runs multiple times, and the service() method is called many times
